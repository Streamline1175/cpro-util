# Centerpiece Pro ‚Äî UE 5.x Skin Template

A minimal Unreal Engine 5.x project for authoring Centerpiece Pro skins.

**Read this first**: Finalmouse has not yet published the SkinEngine SDK. That
means there is no public runtime, no documented skin package format, and no
on-device interactivity bridge. This template is *not* a shipping pipeline. It
is a **staging kit**: a place to author visuals + logic now so you can either

1. bake them to a compliant **video skin** today (works on-device ‚Äî just a .mp4), or
2. port the same assets into Finalmouse's real SDK when it ships.

Anything claimed as "interactive" in this template is a placeholder that
targets assumptions about the future SDK.

## Requirements

- Unreal Engine 5.4 or newer
- Python plugin (included via the `.uproject`)
- Movie Render Pipeline plugin (included)
- A working `cpro` CLI (see repo root README)

## Layout

```
CpSkinTemplate.uproject
Config/
  DefaultEngine.ini      # viewport/render defaults
  DefaultGame.ini
  DefaultInput.ini       # placeholder key mappings (Q/W/E/R/T + LeftShift)
Python/
  setup_skin_project.py  # one-time scaffolder
  render_skin.py         # headless MovieRenderQueue invocation
Content/                 # empty ‚Äî assets generated by setup script
```

## First-time setup

```bash
cpro ue init ~/skins/my-skin
# opens: ~/skins/my-skin/CpSkinTemplate.uproject
```

1. Open the project in UE 5.4+. Let it compile shaders once.
2. Window ‚Üí Developer Tools ‚Üí **Python Console**.
3. Run:
   ```
   py setup_skin_project.py
   ```
   This creates:
   - `/Game/Skin/L_Skin` ‚Äî an empty level
   - `/Game/Skin/BP_SkinActor` ‚Äî Blueprint with stub events (OnKeyDown/Tick etc.)
   - `/Game/Skin/Cine/CineCam_Skin` ‚Äî orthographic cinematic camera, 1920√ó550 filmback
   - `/Game/Skin/Cine/LS_Skin` ‚Äî LevelSequence at 60fps, 3s
4. Open `BP_SkinActor`. Wire Tick + the `Skin_Key_*` action mappings to whatever
   logic you want. These will become real Centerpiece key events when the SDK
   ships ‚Äî wiring them now means a clean port later.
5. Open `LS_Skin` in Sequencer and author your animation.
6. Drag `CineCam_Skin` into your level if needed; make sure it's the active
   camera in the sequence.

## Preview inside UE

- **Viewport aspect**: set the viewport resolution override to 1920√ó550, or use
  the CineCam_Skin preview to see the true crop.
- **PIE**: Play in editor ‚Äî Blueprint stubs will fire off your desk keyboard
  (Q/W/E/R/T) as placeholders for Centerpiece analog keys.

## Export to a compliant video skin

From the repo root (or anywhere `cpro` is on PATH):

```bash
cpro ue export ~/skins/my-skin -o neon.skin.mp4
```

This:

1. invokes `UnrealEditor-Cmd` on your project with `Python/render_skin.py`
2. `render_skin.py` runs MovieRenderQueue at **1920√ó550 @ 60fps**, Apple ProRes `.mov`
3. Node pipes the `.mov` through ffmpeg to produce a spec-compliant `.mp4`
   (H.264 Main@L4.2, `yuv420p`, `+faststart`, audio stripped, 5‚Äì10 Mbps clamped)

Output is a real skin `.mp4` you can upload through
[xpanel.finalmouse.com](https://xpanel.finalmouse.com). Non-interactive ‚Äî the
video is baked ‚Äî but fully usable today.

## Interactive export ‚Äî blocked until Finalmouse releases the SDK

Claims in the Finalmouse docs:

> We are working on a Software Development Kit (SDK) to allow developing your
> own interactive skins, running on our SkinEngine based on Unreal Engine 4.

When it ships, the gap is filled by three unknowns that **only Finalmouse can provide**:

| Unknown | What it unlocks |
| --- | --- |
| On-device skin package format | A packaging target other than "video" |
| Key-event bridge (analog switch data ‚Üí game thread) | Real input wired to your BP stubs |
| Validation/signing scheme | Ability to load your skin on production firmware |

This template deliberately **does not invent** any of those. When the SDK drops,
the porting work is: wire BP_SkinActor's input events to the real bridge, swap
the MovieRenderQueue export for the SDK's packager, and done.

## Known rough edges

- The `CineCam_Skin` actor is placed in the default empty level the first time
  you save the level. If you never save, it disappears on editor close. Save
  the level after running `setup_skin_project.py`.
- `setup_skin_project.py` is idempotent but does not update existing assets ‚Äî
  delete them by hand if you want it to regenerate.
- The render script writes to `Saved/SkinExport/skin_source.mov`. It is deleted
  on each export.
