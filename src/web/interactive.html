How to deploy this as a real interactive skin on your keyboard ▸

What "interactive" actually means

The Record preview skin button above exports a static looping video (.skin.mp4) — the keyboard just plays it on repeat with no reaction to keypresses. A true interactive skin is a packaged Unreal Engine 4.27 app that runs on the keyboard's embedded GPU and fires particle effects in real time when you type. This editor lets you design and preview those effects; the steps below deploy them.

1One-time prerequisites

~15–30 GB, Windows recommended

  • Unreal Engine 4.27.2 — source build from Epic's GitHub (not the launcher version). Run Setup.bat then build the UE4 target in Visual Studio 2019.
  • Visual Studio 2019 with workloads: .NET Desktop, Desktop development with C++, Game development with C++. VS 2022 causes UE 4.27 friction — avoid it.
  • Android Studio — open SDK Manager and install: SDK Platform Android 11 (API 30), Build-Tools 30.0.3, NDK 21.x, CMake 3.18.1, Command-line Tools 8.0.

2Scaffold the project

Run in the same terminal you use for npm run serve

cpro ue pak init ./my-skin

This copies the UE 4.27.2 interactive template into ./my-skin. Then:

  1. Right-click CpInteractiveSkin.uproject → Switch Unreal Engine Version → pick your source build
  2. Open the generated .sln in VS 2019 and build the CpInteractiveSkin project
  3. Open CpInteractiveSkin.uproject in the UE 4.27.2 editor (shader compilation takes 10–30 min on first open)

3Run the scaffolder & author your skin

In the UE editor Python console (Window → Developer Tools → Python):

import unreal, sys
sys.path.insert(0, unreal.Paths.project_dir() + "/Python")
import setup_interactive; setup_interactive.main()

This creates /Game/MySkin/BP_KeyHighlighter and NS_KeyHit. Open BP_KeyHighlighter and wire it:

[On Key Pressed]  (receives: int32 KeyIndex)
  └→ GetPositionByKeyIndex(KeyIndex)
       └→ Break Vector2D  →  X, Y
            └→ Make Vector (X=X-960, Y=Y-275, Z=0)
                 └→ Spawn System At Location (NS_KeyHit)

Customise NS_KeyHit in the Niagara editor to match the colours/particle counts you previewed here. Keep burst count ≤ 30 and Sim Target = CPU Sim.

4Cook, package & upload to the keyboard

From the terminal (requires UE427_ROOT env pointing at your UE source build):

# Cook assets → dist/skin.pak
cpro ue pak cook ./my-skin

# Upload to keyboard slot 0 (auto-discovers on LAN)
cpro ue pak upload dist/skin.pak --slot 0

# If auto-discovery fails, pass the keyboard IP directly
cpro ue pak upload dist/skin.pak --slot 0 --host 192.168.1.42

Slots 0–4 correspond to the five skin slots visible in the Finalmouse XPanel app. The first cook triggers a full Android build inside the editor and can take 30–60 min; subsequent cooks are incremental and much faster.

Need help? Check ue-interactive-template/README.md in the project folder for the full reference.
